# 模型资源

注意

请注意，Collada模型生成与Blockbench模型解析有较大区别，请仔细阅读本章后再进行操作！

# 生成资源包

└─ plugins
   └─ OrangeEngine
      ├─ blockbench               <-  待转换Blockbench模型
      └─ collada                  <-  待转换Collada模型
         └─ 模型名
            └─ 模型文件.dae        <-  待转换Collada模型

如果您想使用一个新的Collada模型，您需要先在OrangeEngine/collada内新建一个自定义的以模型名为名称的文件夹，该文件夹必须为英文，并务必遵循[a-z0-9/._-]格式。

您需要将所有相同模型但动画不同的Collada模型文件放入该文件夹内，模型文件名也请遵循[a-z0-9/._-]格式。

使用指令/oe resource collada即可导出。

# 资源包目录

└─ plugins
   └─ OrangeEngine
      └─ resources         <-  导出资源包
         ├─ pack.mcmeta
         └─ assets
            └─ orangeengine           <-  资源包命名空间不能修改
               └─ collada             <-  模型文件存储位置
                  ├─ model            <-  模型文件存储位置
                  ├─ animation        <-  动画文件存储位置
                  ├─ armor            <-  盔甲模型文件存储位置
                  └─ textures         <-  贴图文件存储位置
                     └─ armor         <-  盔甲贴图文件存储位置
