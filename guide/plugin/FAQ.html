# 常见问题

# 单个bb模型使用多个贴图文件

OrangeEngine暂时不支持单模型使用多个贴图文件，您需要手动Blockbench打开对应的模型文件进行处理。

若不进行处理，可能造成实体纹理渲染错误

# 操作步骤

  1. 在文件 - 插件中打开插件窗口

  1. 搜索名为 Texture Stitcher(McHorse 制作) 的插件并安装

  1. 安装了插件后在纹理区，右键贴图，点击Stitch all textures按钮即可合并为一张

  1. 保存工程文件，重新使用OrangeEngine生成资源文件

# 动态贴图

若在Blockbench中见到以下标识，则说明本模型的贴图文件为动态贴图，您需要手动处理该贴图并替换至资源包

# 操作步骤

  1. 将贴图从Blockbench中导出（点击上图右侧的图标即可保存），或使用OrangeEngine自动导出的贴图。

  2. 将贴图导入到PhotoShop或其他图像处理软件，将图片裁剪仅剩1部分（例如一个动态贴图有两帧，那么他会有上下两部分的贴图拼接而成，此时任意保留一部分即可，通常贴图长宽为2^n）

  3. 替换资源包内贴图

  4. (可选) 替换至Blockbench工程内

# 模型包子模型未正常播放动画

以Hana模型包中的deer_spirit做示例

不做任何修改的情况下，Hana召唤deer_spirit后该模型不会播放任何动画。

# 修复方法

在插件目录OrangeEngine/model内找到该模型的配置文件deer_spirit.yml

以下为默认生成的配置文件

Model:
  Name: deer_spirit
  Hitbox:
    Width: 1.0
    Height: 1.5
  Animation:
    Idle: ''
    Walk: ''
    Jump: ''
    Death: ''

# 子服模型无法正常读取

对于其他子服，您需要在解析模型后手动同步OrangeEngine/data/ER49FqeYQzIYOkltZ8dZtz5DcmI=.oe文件以及OrangeEngine/data/bone中的骨骼数据缓存。
